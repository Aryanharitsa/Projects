For every pair `(a, b)` with `cosine(a, b) ≥ floor` (default `0.18`, tuned slightly above τ so tensions are pulled from the *connected* graph not random buzzword collisions), it runs four detectors and emits a `Tension` when any fire with non-trivial weight: (1) **polarity** — hand-tuned valence lexicons (POSITIVE `good · best · ship · wins · works · scale · …`, NEGATIVE `bad · worst · breaks · fails · overrated · slow · …`) plus a one-content-word negation window that flips local valence ("not bad" → positive, "not great" → negative) so the naive bag-of-words doesn't get reversed by hedging; fires when `sign(pol_a) ≠ sign(pol_b)` *and* `min(|pol_a|, |pol_b|) ≥ 1` so a single polar word per side doesn't trip the rule, weight = `min(1, span/6)`; (2) **antonym** — a 28-pair lexicon (simple/complex, fast/slow, overrated/underrated, robust/brittle, durable/fragile, …) checked against each side's word set, weight saturates at two pairs because contradictions aren't additive; (3) **contrast** — both-sided contrast cues (but, however, although, instead, rather, actually, …); a "but" on *one* side is just writing, a "but" on *both* means each is qualifying the other's frame, weight = `min(1, 0.3 + 0.15·(min(ca,cb)−1))`; (4) **title-clash** — explicit "Against X" vs "Why X works" with light singular/plural stemming (`folder ≈ folders`, the most common plural that originally broke obvious pairs) and stopword filter (so "the" doesn't crown a false match), *plus* opposite-polarity titles with a shared non-polar topic token. The **title-clash bypass** lets unambiguous title contention surface even *below* the cosine floor — the hashing-trick embedder is what it is, and a deterministic linguistic signal shouldn't be overridden by `cosine = 0.04`. `magnitude = clip(cosine · (1 + Σ weights), 0, 1)` so cosine sets the ceiling and signals scale within it. Each emitted `Tension` carries: `kind ∈ {internal, cross}` (same-cluster — where you expected agreement and got conflict — vs philosophical disagreement across topics), the firing `signals[]` with per-signal `(kind, weight, detail)`, two `evidence` quotes (one per side, the **most-polarized sentence in each note in the direction of that note's own stance** — picked via per-note pre-pass over sentence-split bodies, fallback to opening sentence for neutral framings), and a `bridge_title + bridge_prompt + bridge_tags` triple auto-generated from the firing signal kinds ("When X applies vs when Y" for antonym pairs, "Reconciling: X ↔ Y" for polarity, "Bridge: X & Y" otherwise) so **Reconcile** is one click. **Two new endpoints**: `GET /tensions?floor=&limit=&threshold=&top_k=` returns the full report (threshold · floor · total_pairs_scanned · candidate_count · tension_count · tensions[] · stats with internal/cross counts and top_magnitude), `GET /tensions/export.md` ships the same report as a portable artifact (sectioned by Inside-a-cluster / Across-clusters, both quotes with `↑/↓` arrows, signal line, **Reconcile:** prompt) — paste into Notion / journal / anywhere.
